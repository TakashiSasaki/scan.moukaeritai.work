# 画像提供機能の要求仕様 (Image Provisioning Specifications)

本ドキュメントは、アイテム作成・編集フォームにおける「メイン写真 (MAIN PHOTO)」および「周辺写真 (SURROUNDINGS)」の追加方法に関する仕様を定義します。

## 1. 共通仕様
- **対象スロット**: 「メイン写真（1枚）」および「周辺写真（複数枚可）」の2種類。
- **フィードバック**: アップロード中はプログレスインジケータ（スピンアニメーション）を表示する。
- **データ不整合防止**: アップロード完了まで「Save」ボタンなどの操作を制限、または完了後に状態を更新する。
- **エラーハンドリング**: 画像以外のファイルや読み込み失敗時にはトースト通知等でユーザーに知らせる。

## 2. デスクトップ表示 (PC向け)
PC環境では、マウス操作の利便性と大画面を活かした直感的な操作をサポートする。

- **ドラッグ&ドロップ**:
  - 画像ファイルをブラウザ外から直接エリアにドロップしてアップロードできる。
  - ドラッグ中は対象エリアのボーダーカラーや背景色を変更（ハイライト）し、ドロップ可能であることを視覚的に示す。
- **ホバーメニュー**:
  - マウスカーソルを画像エリアに重ねた際、オーバーレイメニューを表示する。
  - 「Upload（ファイル選択ダイアログ）」および「Take Photo（ウェブカメラ起動）」の選択肢を提供する。
- **クリック操作**:
  - エリアをクリックすることで、ファイル選択ダイアログを直接開く、またはメニューを固定表示する。

## 3. モバイル表示 (スマートフォン・タブレット向け)
モバイル環境では、タッチ操作の特性とOS標準のカメラ/写真ライブラリ連携を重視する。

- **タップでのメニュー選択**:
  - 画像エリアをタップすると、メニューをオーバーレイで表示する（ホバーが使えないため、タップで表示状態をトグルする）。
  - メニュー以外の場所をタップすると、メニューを閉じる。
- **カメラ連携**:
  - 「Take Photo」を選択した場合、OS標準のカメラアプリを起動（`capture="environment"`）し、リアカメラ（背面カメラ）を優先して使用する。
- **ライブラリ連携**:
  - 「Upload」を選択した場合、デバイス内の写真ライブラリまたはファイルブラウザから画像を選択できる。

## 4. 技術実装の注意点
- **入力要素の分離**: `capture` 属性を持つインプット（カメラ用）と、持たないインプット（ファイル選択用）を使い分けることで、ユーザーが意図したアクションを確実に実行できるようにする。
- **参照用属性**: `img` タグには `referrerPolicy="no-referrer"` を付与し、Firebase Storage等からの読み込みを確実にする。
